# Tool / Utility Project Type

## Extra folder

Adds `examples/` folder for usage examples and sample configs.

```
project/
└── examples/
    ├── basic-usage.md
    ├── advanced-usage.md
    └── sample-configs/
```

## CLAUDE.md additions

Add this section to the project's CLAUDE.md after Task Routing:

```markdown
### Working on tool features or examples
1. Read `docs/architecture/MODULE-OVERVIEW.md`
2. Read `project/examples/` for current usage patterns
```

## Initial doc stubs

When scaffolding a tool project, create these starter docs:
- `docs/context/TOOL-PURPOSE.md` (placeholder)
- `docs/architecture/MODULE-OVERVIEW.md` (placeholder)
- `docs/workflows/CONTRIBUTING.md` (placeholder)
